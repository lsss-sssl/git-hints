## Основные команды Git (для ежедневной работы)

### Инициализация и клонирование
- `git init` – создать новый локальный репозиторий
- `git clone <url>` – клонировать существующий репозиторий на компьютер

### Работа с файлами и коммитами
- `git status` – показать состояние рабочей директории и индекса
- `git add <file>` – добавить файл в индекс
- `git add .` – добавить все изменения в индекс
- `git restore <file>` – отменить локальные изменения файла
- `git restore --staged <file>` – убрать файл из индекса, сохранив изменения
- `git rm <file>` – удалить файл и сразу добавить удаление в индекс
- `git commit -m "сообщение"` – создать коммит
- `git commit --amend --no-edit` – изменить предыдущий коммит без изменения сообщения

### Просмотр изменений
- `git diff` – показать изменения в рабочей директории
- `git diff --staged` – показать изменения, которые попадут в коммит

### Ветки
- `git branch` – список веток
- `git branch <name>` – создать новую ветку
- `git switch <name>` – переключиться на существующую ветку
- `git switch -c <name>` – создать и переключиться на новую ветку
- `git merge <branch>` – объединить выбранную ветку с текущей
- `git rebase <branch>` – перенести коммиты на другую ветку для чистой истории

### Работа с удалёнными репозиториями (GitHub)
- `git remote -v` – показать подключённые удалённые репозитории
- `git pull` – получить и объединить изменения
- `git push` – отправить коммиты на GitHub
- `git fetch` – получить изменения без объединения

### История и откат изменений
- `git log` – показать историю коммитов
- `git show <hash>` – показать детали коммита
- `git reset --soft HEAD~1` – отменить последний коммит, оставив изменения в индексе
- `git reset HEAD~1` – отменить последний коммит, сохранив изменения в рабочей директории
- `git reset --hard <hash>` – откатить проект к выбранному коммиту (опасно)
- `git revert <hash>` – создать новый коммит, отменяющий изменения выбранного (безопасно для GitHub)
- `git checkout <hash>` – перейти на конкретный коммит (detached HEAD)

### HEAD
- `HEAD` – текущий коммит
- `HEAD~1` – один коммит назад
- `HEAD~2` – два коммита назад
