# Использование Git-команд

Этот документ объясняет, как применять Git-команды в типичном рабочем процессе, включая создание веток, внесение изменений, проверку, слияние и развертывание.

---

## 1. Подготовка рабочей среды

Перед началом работы необходимо:

* Клонировать репозиторий: `git clone <url>`
* Перейти в созданную директорию проекта

---

## 2. Создание рабочей ветки

Работа над задачей всегда выполняется в отдельной ветке.

* Создать новую ветку: `git switch -c <branch>`
* Убедиться, что ветка создана и выбрана: `git branch`

---

## 3. Внесение изменений

Изменяем файлы, затем выполняем:

* Проверить статус файлов: `git status`
* Добавить файлы в индекс: `git add <file>` или `git add .`
* Создать коммит: `git commit -m "сообщение"`

---

## 4. Синхронизация с удалённой веткой

Чтобы изменения были доступны на сервере:

* Отправить ветку: `git push -u origin <branch>`
* При последующих пушах: `git push`

---

## 5. Обновление ветки перед merge

Чтобы избежать конфликтов:

* Переключиться в основную ветку: `git switch main`
* Получить новые изменения: `git pull`
* Перейти назад в свою ветку: `git switch <branch>`
* Перенести изменения:

  * merge: `git merge main`
  * или rebase: `git rebase main`

---

## 6. Создание Pull Request (PR)

На GitHub:

1. Открыть вкладку Pull Requests
2. Создать PR из своей ветки в основную
3. Дождаться ревью

---

## 7. Завершение работы

После одобрения PR:

* PR сливается в основную ветку
* Локально обновить основную ветку: `git switch main` → `git pull`
* Удалить рабочую ветку: `git branch -d <branch>`
* Удалить её на GitHub: `git push origin --delete <branch>`

---

## 8. Исправление ошибок

* Вернуть изменения в файле: `git restore <file>`
* Убрать файл из индекса: `git restore --staged <file>`
* Отменить последний коммит:

  * мягко: `git reset --soft HEAD~1`
  * полностью: `git reset --hard HEAD~1`
* Безопасный откат коммита: `git revert <hash>`

---

## 9. Просмотр истории и диагностика

* История изменений: `git log`
* Просмотр конкретного коммита: `git show <hash>`
* Просмотр различий: `git diff`
* Просмотр того, что попадёт в коммит: `git diff --staged`
